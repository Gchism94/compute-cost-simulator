Read prompts/00_project_context.md first.

Add more realistic usage scenarios showing how simulated compute costs differ between structured classroom use and more uninhibited student project use.

Goal:
Help students and instructors see that:
- in-class labs often use small/simple models and constrained examples
- independent projects often use larger models, more iterations, more AI help, and RAG workflows
- small per-action costs can creep quickly when repeated outside class
- classroom-scale costs are different from individual project costs

Tasks:

1. Add a new example script:

examples/realistic_usage_scenarios_demo.py

This script should simulate and compare at least three scenarios:

A. Structured In-Class Lab
- simple models
- limited runs
- small LLM/token use
- constrained budget
- example actions:
  - logistic regression
  - random forest
  - small prompt for explanation/debugging

B. Student Project Workflow
- more sophisticated models
- more experimentation
- more AI assistant usage
- example actions:
  - random forest or XGBoost-like simulated model
  - neural network
  - transformer/large model simulation
  - several debugging/refactor AI calls
  - small RAG retrieval session

C. Uninhibited AI-Assisted Workflow
- repeated chatbot use
- verbose prompts
- pasted tracebacks
- RAG context
- multiple retries
- more expensive model size
- many token-heavy calls

For each scenario, show:
- total cost
- action count
- AI/LLM cost
- modeling cost
- percentage of cost from AI
- projected classroom cost for 40 students
- projected repeated-project cost if each student repeats the workflow 5 times

Use existing CCS API:
- Budget
- compute_block
- track_llm_call
- summarize_logs

Use time.sleep() to simulate compute.
Do not add external dependencies.

2. Make the output student-facing.

Use headings like:
- Scenario A: Structured In-Class Lab
- Scenario B: Student Project Workflow
- Scenario C: Uninhibited AI-Assisted Workflow
- What to notice
- Questions to consider

Avoid instructor-facing phrasing such as:
- What students should notice

3. Make the cost contrast obvious.

The in-class lab should be cheap.
The student project should be noticeably more expensive.
The uninhibited workflow should show costs creeping quickly due to repeated AI calls and larger models.
